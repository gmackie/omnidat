# Drizzle ORM Migration Guide

This project uses [Drizzle ORM](https://orm.drizzle.team/) with Neon Postgres for type-safe database access and migrations.

## Quick Reference

```bash
# Generate a migration from schema changes
pnpm --filter @gmacko/db generate

# Apply pending migrations to the database
pnpm --filter @gmacko/db migrate

# Push schema directly (development only - skips migration files)
pnpm --filter @gmacko/db push

# Open Drizzle Studio (visual database browser)
pnpm --filter @gmacko/db studio

# Check migration status
pnpm --filter @gmacko/db check

# Seed the database with default accounts and sample data
pnpm --filter @gmacko/db seed
```

## Workflow

### 1. Modify Your Schema

Edit `packages/db/src/schema.ts` or `packages/db/src/auth-schema.ts`:

```typescript
// packages/db/src/schema.ts
export const myNewTable = pgTable("my_new_table", (t) => ({
  id: t.uuid().notNull().primaryKey().defaultRandom(),
  name: t.varchar({ length: 255 }).notNull(),
  createdAt: t.timestamp().defaultNow().notNull(),
}));
```

### 2. Generate Migration

```bash
pnpm --filter @gmacko/db generate
```

This creates a new SQL migration file in `packages/db/drizzle/`. Review it before applying.

### 3. Apply Migration

```bash
# To your development database
pnpm --filter @gmacko/db migrate

# Or push directly (dev only, no migration file)
pnpm --filter @gmacko/db push
```

### 4. Update Seed Script

If your new table needs sample data, update `packages/db/src/seed.ts`.

## Environment-Specific Databases

| Environment | Database | Connection |
|-------------|----------|------------|
| Local (Docker) | PostgreSQL 16 via docker-compose | `postgresql://postgres:postgres@localhost:5432/gmacko_dev` |
| Local (PGlite) | Embedded PG, no server needed | `DATABASE_DRIVER=pglite` |
| Staging | Neon Postgres (staging branch) | `STAGING_DATABASE_URL` |
| Production | Neon Postgres (main branch) | `DATABASE_URL` |

### Local Development with Docker

```bash
# Start PostgreSQL
docker compose up postgres -d

# Push schema
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/gmacko_dev" \
  pnpm --filter @gmacko/db push

# Seed with default accounts
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/gmacko_dev" \
  pnpm --filter @gmacko/db seed
```

### Local Development with PGlite (Zero Dependencies)

```bash
# No Docker or external database needed
DATABASE_DRIVER=pglite pnpm dev
```

PGlite runs an embedded PostgreSQL in-process, persisted to `.local/pglite/`.

## Running Migrations in CI/CD

The `migrations.yml` GitHub workflow handles migrations automatically:

```yaml
# Triggered on push to main (production) or staging branches
# Uses environment-specific DATABASE_URL secrets
```

### Manual CI Migration

```bash
# Staging
DATABASE_URL=$STAGING_DATABASE_URL pnpm --filter @gmacko/db migrate

# Production
DATABASE_URL=$PRODUCTION_DATABASE_URL pnpm --filter @gmacko/db migrate
```

## Schema Organization

```
packages/db/src/
├── auth-schema.ts    # Better Auth tables (user, session, account, verification)
├── schema.ts         # Application tables (post, subscription, purchase, preferences, apiKeys)
├── client.ts         # Database connection (auto-selects Neon/PG/PGlite)
├── client-local.ts   # PGlite local development client
├── seed.ts           # Seed script with default accounts
└── index.ts          # Re-exports
```

## Tips

- **Always generate migrations** in production workflows. Never use `push` in staging/production.
- **Review generated SQL** before applying. Drizzle generates migrations but they may need manual adjustment for data migrations.
- **Use `casing: "snake_case"`** in drizzle config to keep column names consistent with PostgreSQL conventions.
- **Drizzle Studio** (`pnpm db:studio`) is great for inspecting data during development.
- **Keep auth schema separate** — it's generated by Better Auth CLI and shouldn't be manually edited.
