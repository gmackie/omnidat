# Assimilation Protocol

All activity enters the record.

## Process

1. Observation begins at first packet or terminal report.
2. Clearance is issued or withheld according to protocol.
3. Every termination is logged with cause and diagnostic.
4. The ledger updates. The file is never closed.

## Correction Vectors

- Brevity enforcement via session accounting.
- Namespace boundaries enforced by address allocation.
- Budget limits on user data per transport.
- Operator vectors execute correction when deviation exceeds threshold.

## Outcome

Compliance registers as normal operation. Deviation registers as event. Events are permanent.

There is no external state. All business is already inside the system.
